# 🚀 시스템 심층 기술 명세 최종본: Retraining & RAG 정밀화

본 문서는 제안된 시스템의 기술적 유효성을 완벽히 입증하기 위해, 파이프라인의 각 단계별 알고리즘, 수학적 정당성, 그리고 구현 파라미터를 극도로 상세하게 기술합니다.

---

## 1. 정교화된 RAG(Retrieval-Augmented Generation) 파이프라인

리뷰어의 "상세 설명 미흡" 지적을 해결하기 위해, 인출(Retrieval)부터 생성(Generation)까지의 내부 로직을 최적화 파라미터와 함께 명시합니다.

### 1.1 하이브리드 인출(Hybrid Retrieval) 메커니즘
본 시스템은 벡터 유사도 검색의 한계를 극복하기 위해 **Reciprocal Rank Fusion (RRF)** 알고리즘을 사용하여 검색 결과를 재정렬한다.

- **BM25 점수 (키워드)**: 식당명, 주요 메뉴 등 핵심 키워드의 출현 빈도 계산.
- **벡터 점수 (의미)**: KoSimCSE를 통한 쿼리와 리뷰 간의 코사인 유사도 계산.
- **RRF 공식**: 
  $$Score(d) = \sum_{r \in R} \frac{1}{k + rank(r, d)}$$
  ($k$는 상수, 보통 60 사용)
- **효과**: 사용자의 의도(의미)와 특정 명사(키워드) 대응 능력을 동시에 확보하여 검색 정밀도(Precision)를 극대화함.

### 1.2 지식의 구조화 및 청킹(Chunking) 전략
- **Chunk Size**: 블로그 리뷰 데이터를 512 토큰 단위로 분할하여 컨텍스트 윈도우 내 정보 밀도를 최적화.
- **Overlap**: 인접 청크 간 50 토큰의 중첩을 두어 문장 끊김으로 인한 문맥 손실(Context Loss) 방지.
- **Top-K Selection**: $K=5$로 설정. 이는 정보의 과잉(Information Overload)을 방지하고 LLM의 응답 신뢰도를 높이는 최적의 수치임.

## 2. 피드백 기반 소프트 재학습(Soft-Retraining) 상세 알고리즘

"재학습 구조의 구체성"을 입증하기 위해 데이터 갱신 및 가중치 업데이트 알고리즘을 설명한다.

### 2.1 지수 이동 평균(EMA) 기반 평판 업데이트
단순 평균은 과거 데이터에 편향될 수 있으므로, 최신 트렌드를 반영하기 위해 **지수 이동 평균(EMA)** 방식을 사용하여 장소의 평판 점수($S$)를 관리한다.

- **업데이트 수식**: $S_t = \alpha \cdot R_t + (1 - \alpha) \cdot S_{t-1}$
  ($R_t$: 새로운 별점, $\alpha$: 최신 데이터 가중치, 보통 0.2~0.3 사용)
- **이점**: 최근 식당의 서비스 품질 저하나 인테리어 개선 등의 변화를 시스템이 빠르게 '학습'하여 추천 결과에 반영함.

### 2.2 비동기 피드백 처리 파이프라인 (The Back-end Loop)
- **Celery & Redis Orchestration**: 
  - 사용자의 별점 및 GPS 체류 시간 데이터가 발생하면 즉시 Redis 큐에 적재.
  - Celery Worker가 비동기적으로 해당 데이터를 분석하고 이상치(Outlier, 예: 1분 미만 체류 등)를 제거.
  - 가공된 정보를 Supabase PostgreSQL의 JSONB 필드(`meta.duration_by_condition`)에 원자적(Atomic)으로 업데이트.
- **In-Context Injection**: 다음 호출 시 RAG 엔진은 갱신된 `meta` 정보를 기반으로 프롬프트를 동적 생성. 이는 신경망 가중치를 고정(Frozen)한 채 시스템의 반응(Response)을 영구적으로 변경시키는 **'온라인 학습'** 효과를 창출함.

## 3. 구현 상세 및 하이퍼파라미터 (Implementation Specs)

학술적 객관성 확보를 위해 실제 적용된 시스템 파라미터를 공개한다.

| 파라미터 | 설정값 | 이유 |
| :--- | :--- | :--- |
| **Embedding Model** | KoSimCSE-roberta-multitask | 한국어 문장 의미 추출 성능 최우수 |
| **Vector Dimension** | 768-D | 모델 출력값과 1:1 대응하여 손실 최소화 |
| **Indexing** | HNSW (M=16, ef_construction=64) | 실시간성(ms 단위)과 정확도의 최적 균형점 |
| **LLM Temperature** | 0.2 | 추천 결과의 일관성 및 할루시네이션 방지 |
| **Similarity Threshold** | 0.75 | 유의미한 관련성을 가진 컨텍스트만 선별 |

## 4. 정량적 실험 설계 및 품질 보증 (Evaluation)

### 4.1 지표 정의
1. **Precision@5**: 추천된 5개 장소 중 사용자가 실제 방문하거나 긍정적으로 평가한 비율.
2. **Contextual Relevance Score**: LLM이 생성한 추천 사유가 인출된 블로그 리뷰 내용과 얼마나 일치하는지 인간 평가자가 5점 척도로 측정.
3. **Latency**: 사용자 요청부터 최종 코스 생성까지의 전체 소요 시간 (목표: 3초 이내).

### 4.2 자가 학습 유효성 검증
피드백 루프 적용 전(Zero-Iter)과 적용 후(N-Iter)의 사용자 만족도 변화를 측정하여, **소프트 재학습 구조가 실제 추천 품질을 지속적으로 향상시킴**을 통계적 유의성($p < 0.05$)으로 입증함.
