# 📑 2026 스마트미디어학회 논문 보완: 학술적 상세 명세 (Retraining & RAG)

본 문서는 논문 심사위원의 피드백인 "재학습 구조 및 RAG 파이프라인의 기술적 상세 설명 부족"을 해결하기 위해, 시스템의 핵심 메커니즘을 수학적/기술적 관점에서 상세히 기술합니다.

## 1. RAG(Retrieval-Augmented Generation) 파이프라인 상세 사양

본 시스템은 단순한 정보 검색을 넘어, 사용자의 페르소나와 실시간 지역 데이터를 동적으로 결합하는 고도화된 RAG 파이프라인을 채택한다.

### 1.1 데이터 인코딩 및 임베딩 공간 정의
- **임베딩 모델**: 한국어 문맥 보존 능력이 검증된 `KoSimCSE-roberta-multitask` 모델을 사용한다.
- **벡터화 과정**: 텍스트 데이터(리뷰, 식당 상세정보 등)를 768차원의 밀집 벡터(Dense Vector, $V \in \mathbb{R}^{768}$)로 변환하여 의미적 공간(Semantic Space)에 매핑한다.
- **유사도 측정**: 사용자의 요구사항 벡터($q$)와 데이터베이스 내 벡터($v$) 간의 근접도를 측정하기 위해 다음과 같은 **코사인 유사도(Cosine Similarity)** 수식을 적용한다.
  $$\text{Similarity}(q, v) = \frac{q \cdot v}{\|q\| \|v\|}$$

### 1.2 고속 벡터 검색을 위한 HNSW 인덱싱
- **문제 해결**: 수천 건 이상의 비정형 데이터셋에서 밀리초(ms) 단위의 반응 속도를 확보하기 위해 **HNSW (Hierarchical Navigable Small World)** 그래프 알고리즘을 적용한다.
- **기술적 이점**: 고차원 공간에서 각 노드를 계층적으로 연결함으로써, 전체 데이터를 전수 조사하지 않고도 근사 최접점(Approximate Nearest Neighbor)을 빠르게 찾아내어 실시간 추천의 유효성을 보장한다.

### 1.3 하이브리드 리트리벌 (Hybrid Retrieval) 전략
- **과정**: 단순 벡터 검색의 한계를 극복하기 위해, 사용자의 지리적 위치(GPS) 및 이동수단에 따른 **메타데이터 필터링**을 선행한 후, 남은 후보군 내에서 상위 K개의 유사 리뷰 컨텍스트를 LLM에 전달한다.

## 2. 피드백 기반 재학습 구조 (Soft-Retraining Loop)

전통적인 딥러닝 모델의 재학습(Full Fine-tuning)은 막대한 비용과 시간이 소요되어 실시간 추천 시스템에는 부적합하다. 본 시스템은 이를 극복하기 위해 **소프트 재학습(Soft-Retraining)** 매커니즘을 도입한다.

### 2.1 인-컨텍스트 피드백 루프 (In-Context Feedback Loop)
본 시스템의 '재학습'은 모델의 파라미터를 직접 수정하는 대신, **지속적으로 업데이트되는 외부 지식(Context)을 동적으로 주입하여 모델의 출력 결과물을 강화**하는 방식이다.

1.  **실시간 피드백 수집**: 사용자가 제출한 별점 정보와 GPS 로그를 통해 집계된 관광지별 실제 체류 시간(Residence Time)을 상시 수집한다.
2.  **비동기 데이터 어그리게이션 (Celery & Redis)**: 수집된 원시 데이터를 백그라운드에서 분석하여, 특정 동행 유형(예: 아이 동반 가족)에 최적화된 예상 체류 시간과 최근 평판 점수를 도출한다.
3.  **지식 베이스(Vector DB) 동적 갱신**: 분석된 결과는 데이터베이스 내 메타데이터 필드(`meta.duration_by_condition`, `score`)에 즉각 반영되며, 이는 다음 RAG 파이프라인 가동 시 LLM의 프롬프트에 '최신화된 정보'로 주입된다.

### 2.2 메커니즘의 유효성
이 구조를 통해 모델은 물리적인 가중치 업데이트 없이도 실시간 트렌드와 장소의 운영 상태(예: 최근 맛이 변했다는 리뷰 급증 등)를 즉각적으로 '학습'하여 추천 품질을 자가 개선(Self-Improvement)하게 된다.

## 3. 정량적 시스템 평가지표 및 실험 설계

제안된 시스템의 우수성을 입증하기 위해 다음과 같은 정량적 실험 프레임워크를 구성한다.

### 3.1 성능 평가지표 (Metrics)
- **Hit Rate@k**: 전체 추천 경로 중 사용자가 실제로 만족하거나 방문한 장소가 상위 k개 내에 포함된 비율을 측정한다.
- **MRR (Mean Reciprocal Rank)**: 추천된 장소들의 순위 역수를 평균하여, 정답에 가까운 추천이 얼마나 상단에 노출되는지 평가한다.
- **Response Latency**: RAG 파이프라인 가동 시 소요되는 전체 시간을 측정하여 실시간 서비스로서의 적합성을 검증한다.

### 3.2 비교 분석 (Comparative Analysis)
- **Baseline 1**: 기존 위치 기반 규칙(Rule-based) 추천
- **Baseline 2**: RAG 가 적용되지 않은 범용 LLM 추천
- **Proposed**: 위치 정보와 유기적 리뷰 RAG가 결합된 제안 시스템
- **기대 결과**: 제안 시스템은 Baseline 대비 Hit Rate 기준 최소 20% 이상의 성능 향상을 목표로 하며, 이는 RAG를 통한 지식 보강과 피드백 루프의 유효성을 실증한다.
